# MediMind - Healthcare RAG Question Answering Platform
A modern, full-stack Retrieval-Augmented Generation (RAG) application for answering healthcare and insurance-document questions with evidence-grounded responses.  
Built with **React + TypeScript + Vite** (frontend) and **FastAPI + Python** (backend), powered by **FAISS/ChromaDB**, **sentence-transformer embeddings**, and **Ollama LLMs**.
---
## ✨ Features
### 🎯 Core Features
- **Document Q&A**: Upload PDF/TXT/MD and ask natural language questions
- **Evidence-grounded responses**: Answers include retrieved source chunks
- **Strict prompt mode**: Document-only answering for high-trust workflows
- **Session-based ingestion**: Each uploaded document creates an isolated QA session
- **Streaming chat**: Token streaming via SSE for smooth response UX
- **Source transparency**: View source rank, page, chunk size, score, and context block index
- **Evaluation mode**: Run built-in benchmark and compare configurations
### 🛠 Technical Features
- **Multiple chunking strategies**: fixed, sentence-based, semantic, by-page
- **Multiple embeddings**: MiniLM, BGE-small, MPNet (and optional OpenAI embedder path)
- **Two vector databases**: FAISS and ChromaDB
- **Hybrid retrieval**: dense retrieval + optional cross-encoder rerank
- **Boilerplate mitigation**: retrieval penalties for repeated PDF letterhead chunks
- **Diversity selection**: MMR-style source diversification to avoid repetitive evidence
- **FastAPI API layer**: health, ingest, chat, streaming, evaluate, traces, ablation endpoints

🚀 Quick Start
Prerequisites
Python 3.10+
Node.js 18+
Ollama installed and running
(Optional) HF token for gated Hugging Face models in experiment scripts
1) Clone and enter project
git clone <your-project-url>
cd healthcare_rag
2) Backend setup
python -m venv .venv
source .venv/bin/activate        # macOS/Linux
pip install -r requirements.txt
python -c "import nltk; nltk.download('punkt'); nltk.download('punkt_tab')"
3) Pull Ollama model(s)
ollama pull gemma3:12b
ollama pull gemma3:4b
4) Start backend
uvicorn api.main:app --reload --port 8000
5) Start frontend
cd medimind-ui
npm install
npm run dev
Visit: http://localhost:5173

⚙️ Environment Variables
You can run with defaults, but these are supported:

# Ollama defaults
HEALTHCARE_RAG_OLLAMA_MODEL=gemma3:12b
# Hugging Face token for gated models (experiment scripts / HF fallback)
HF_TOKEN=hf_xxx
# Optional fallback model used by experiment scripts
HEALTHCARE_RAG_HF_LLM_FALLBACK=Qwen/Qwen2.5-1.5B-Instruct

🧠 Retrieval + Generation Flow
Ingest: Document is loaded and cleaned
Chunk: Text split by selected strategy
Embed: Chunk vectors generated by selected embedding model
Index: Stored in FAISS or ChromaDB
Retrieve: Fetch-K candidates, then rank to Top-K
Rank: Optional cross-encoder rerank + boilerplate downweight + MMR diversity
Prompt: Numbered context blocks injected into prompt template
Generate: Ollama model returns grounded answer + sources metadata

🔒 Safety and Reliability Notes
Strict template mode minimizes unsupported claims by constraining answers to provided context.
Source cards expose evidence used for answer generation.
Confidence shown is heuristic (retrieval-score based), not clinical certainty.
This system is for educational/research use and should not replace professional advice.
